## Architecture Overview

```
┌─────────────────────┐        HTTP/REST        ┌──────────────────────────┐
│   React Frontend    │ ──────────────────────► │   Express.js Backend     │
│   (Vite, port 5173) │  (proxied via /api)     │   (Node.js, port 3001)   │
└─────────────────────┘                         └──────────┬───────────────┘
                                                           │ AWS SDK v3
                                                           ▼
                                                ┌──────────────────────────┐
                                                │   AWS DynamoDB           │
                                                │   (region: sa-east-1)    │
                                                │                          │
                                                │  Users · Strategies      │
                                                │  Alerts · Subscriptions  │
                                                │  PaymentMethods          │
                                                │  Invoices · Payments     │
                                                └──────────────────────────┘
```

The frontend dev server proxies all `/api` requests to the backend, so there are no CORS issues during local development.

## Features

- **Strategy Management** — Create and configure trading strategies with 40+ technical indicators (RSI, MACD, Bollinger Bands, candlestick patterns, and more), across Crypto, Forex, and Stock symbols.
- **Alerts** — Paginated history of buy/sell signals generated by active strategies.
- **User Profile** — Manage personal information and configure notification channels (Email, SMS).
- **Subscriptions** — Free / Standard / Pro plans with different strategy limits and alert frequencies.
- **Billing** — Manage payment methods (Credit Card, PayPal, Crypto wallet), view invoices, and payment history.
- **Authentication** — Email/password sign-up with email confirmation flow. Passwords hashed with scrypt + random salt.

## Running Locally

### Prerequisites

- Node.js 18+
- An AWS account with DynamoDB tables provisioned (see [schema below](#aws-dynamodb-schema))
- AWS credentials configured (either via `server/.env` or `~/.aws/credentials`)

### 1. Install dependencies

```bash
# Frontend (project root)
npm install

# Backend
cd server
npm install
```

### 2. Configure environment variables

Create `server/.env` (see [Environment Variables](#environment-variables) for the full list):

```bash
cp server/.env.example server/.env
# then fill in your values
```

### 3. Start the servers

Open two terminals:

```bash
# Terminal 1 — Frontend (http://localhost:5173)
npm run dev

# Terminal 2 — Backend (http://localhost:3001)
cd server
npm run dev
```

The Vite proxy forwards all `/api` calls from the frontend to the backend automatically — no extra configuration needed.

## Environment Variables

All backend configuration lives in `server/.env`.

| Variable | Example Value | Description |
|---|---|---|
| `PORT` | `3001` | Express server port |
| `AWS_REGION` | `sa-east-1` | AWS region |
| `USERS_TABLE` | `Users` | DynamoDB Users table name |
| `USERS_USERID_GSI` | `USERS_USERID_GSI` | GSI on Users for userId lookups |
| `STRATEGIES_TABLE` | `Strategies` | DynamoDB Strategies table name |
| `ALERTS_TABLE` | `Alerts` | DynamoDB Alerts table name |
| `ALERTS_USERID_GSI` | `UserIdIndex` | GSI on Alerts for userId queries |
| `SUBSCRIPTIONS_TABLE` | `Subscriptions` | DynamoDB Subscriptions table name |
| `PAYMENT_METHODS_TABLE` | `PaymentMethods` | DynamoDB PaymentMethods table name |
| `INVOICES_TABLE` | `Invoices` | DynamoDB Invoices table name |
| `PAYMENTS_TABLE` | `Payments` | DynamoDB Payments table name |

AWS credentials (`AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`) can be placed in `.env` or managed via `~/.aws/credentials`.

## API Reference

All routes are prefixed with `/api`.

### Auth
| Method | Path | Description |
|---|---|---|
| POST | `/auth/login` | Authenticate user |
| POST | `/auth/signup` | Register new user (creates inactive account) |
| POST | `/auth/confirm/:userId` | Activate account via email confirmation link |
| POST | `/auth/forgot-password` | Initiate password reset |

### Users
| Method | Path | Description |
|---|---|---|
| GET | `/users/:userId` | Get user profile |
| POST | `/users/:userId` | Update user profile |

### Strategies
| Method | Path | Description |
|---|---|---|
| GET | `/strategies/user/:userId` | List strategies (summary) |
| POST | `/strategies/user/:userId` | Create strategy |
| GET | `/strategies/:userId/:strategyId` | Get strategy details |
| POST | `/strategies/:userId/:strategyId` | Update strategy |
| DELETE | `/strategies/:userId/:strategyId` | Delete strategy |

### Alerts
| Method | Path | Description |
|---|---|---|
| GET | `/alerts/user/:userId?limit=10&nextToken=...` | Paginated alert history |

### Subscriptions
| Method | Path | Description |
|---|---|---|
| GET | `/subscriptions/user/:userId` | Get active subscription |
| POST | `/subscriptions/user/:userId` | Create or update subscription |
| POST | `/subscriptions/user/:userId/:subscriptionId/cancel` | Cancel subscription |

### Payment Methods
| Method | Path | Description |
|---|---|---|
| GET | `/payment-methods/user/:userId` | List payment methods |
| POST | `/payment-methods/user/:userId` | Add payment method |
| POST | `/payment-methods/user/:userId/:paymentMethodId/set-default` | Set default method |
| DELETE | `/payment-methods/user/:userId/:paymentMethodId` | Remove payment method |

### Invoices & Payments
| Method | Path | Description |
|---|---|---|
| GET | `/invoices/user/:userId` | List invoices |
| GET | `/invoices/user/:userId/:invoiceId` | Get invoice details |
| GET | `/payments/user/:userId` | List payment records |
